# 本地管理员免登录设计

## 目标

通过本地启动器运行时，直接以数据库中首个启用的管理员身份进入教师工作台；临时公网和其他常规运行方式继续要求账号密码。

## 边界

- `scripts/start_local.sh` 仅为其启动的 Python 进程设置 `ACADEMIC_PLANNING_LOCAL_ADMIN=1`。
- 应用在该变量为真时加载首个启用管理员到 `g.current_user`，跳过登录重定向，并继续执行既有学生访问与角色权限规则。
- 不修改用户表、不写入会话、不改变公网启动脚本。
- 没有启用管理员时，本地请求返回明确的初始化错误，不能匿名放行。

## 验证

- 本地管理员模式可访问受保护的教师页面且 `current_user` 为管理员。
- 普通运行仍重定向到登录页。
- 本地管理员模式没有管理员时拒绝访问。
- 本地启动脚本导出该变量，公网启动脚本不导出。
