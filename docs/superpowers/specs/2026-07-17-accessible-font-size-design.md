# 全站易读字号设计

## 目标

让近视或需要较大字号的使用者能够直接阅读系统，不依赖浏览器缩放；所有界面的普通、辅助、表格与表单文字均不小于 22px，并充分使用大屏幕横向空间。

## 方案选择

- 推荐并采用：在共享样式表中统一设定 22px 的最低字号、24px 的常规字号，并按信息层级放大导航、标签、表头、说明、按钮与输入控件；桌面端改为近全宽内容区。
- 未采用：仅提示用户放大浏览器。该方案无法保证所有用户获得一致体验。
- 未采用：单独制作「大字号模式」开关。当前需求是全站默认清晰易读，增加设置会提高使用复杂度。

## 视觉与交互设计

- 正文和默认继承文字使用 24px；行高提高至 1.65，降低连续阅读负担。
- 辅助信息（字段标签、提示、表格表头、指标说明、导航）使用 22px，不再保留小于该数值的界面文字。
- 页面主标题升至 42px，卡片标题升至 30px，品牌名称升至 28px，保留清晰的信息层级。
- 输入框、选择框和按钮随字体增大：最小高度提高至约 60px，表格单元格和面板的内边距相应增加。
- 桌面端页面容器使用视口宽度减去 64px，取消 1120px 的最大宽度；大屏幕左右保留约 32px 边距，表格与表单按可用宽度展开。
- 手机端仍保持与桌面端相同的最低字号，并将页面边距收敛为 12px，避免内容横向溢出。

## 实施范围

修改 `app/static/styles.css` 的全局组件规则，并在基础模板末尾加入最终的可读性覆盖规则，使页面局部样式也不能将普通文字降到 22px 以下；不改动数据模型或业务逻辑。

## 验收与测试

- 样式回归测试断言基础字号、辅助文字和表格表头均为至少 22px。
- 测试同时断言表单控件和按钮的最小高度已提高，防止出现文字变大而控件拥挤。
- 测试断言桌面页面宽度不再限制为 1120px，并保留适合小屏幕的边距规则。
- 运行全量测试套件，确保样式调整不影响既有功能。
