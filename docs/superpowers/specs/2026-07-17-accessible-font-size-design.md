# 全站易读字号设计

## 目标

让近视或需要较大字号的使用者能够直接阅读系统，不依赖浏览器缩放；所有界面的普通、辅助、表格与表单文字均不小于 16px。

## 方案选择

- 推荐并采用：在共享样式表中统一设定 16px 的基础字号，并按信息层级放大导航、标签、表头、说明、按钮与输入控件。
- 未采用：仅提示用户放大浏览器。该方案无法保证所有用户获得一致体验。
- 未采用：单独制作「大字号模式」开关。当前需求是全站默认清晰易读，增加设置会提高使用复杂度。

## 视觉与交互设计

- 正文和默认继承文字使用 16px；行高提高至 1.6，降低连续阅读负担。
- 辅助信息（字段标签、提示、表格表头、指标说明、导航）使用 16px，不再保留 13px 或 14px 的小字。
- 页面主标题升至 34px，卡片标题升至 24px，品牌名称升至 22px，保留清晰的信息层级。
- 输入框、选择框和按钮随字体增大：最小高度提高至约 48px，表格单元格和面板的内边距相应增加。
- 手机端保留与桌面端相同的最低字号；仅将主标题收敛到 30px，避免窄屏换行过多。

## 实施范围

仅修改 `app/static/styles.css` 的全局组件规则，不改动页面模板、数据模型或业务逻辑，因此所有现有页面自动获得一致的易读字号。

## 验收与测试

- 样式回归测试断言基础字号、辅助文字和表格表头均为至少 16px。
- 测试同时断言表单控件和按钮的最小高度已提高，防止出现文字变大而控件拥挤。
- 运行全量测试套件，确保样式调整不影响既有功能。
